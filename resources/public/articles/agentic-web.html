1. The Steinberger Thesis: APIs Over Protocols

Peter Steinberger — founder of PSPDFKit, veteran open-source developer, and creator of OpenClaw — holds a clear position: agents should call APIs directly, not through bespoke middleware protocols.

"Why invent a new protocol when every service already has an API? Agents should call APIs — the same way developers do."

His reasoning is pragmatic: every service worth integrating already exposes REST endpoints, GraphQL schemas, or CLI tools. These interfaces are documented, battle-tested, and understood by millions of developers. Why add an abstraction layer?

2. The Internet Parallel

The internet succeeded because it built on simple, composable interfaces:

TCP/IP→ HTTP→ HTML/CSS→ Browsers→ The Web

The agentic web follows the same pattern — but instead of inventing new protocols, it builds on the APIs we already have:

TCP/IP→ REST / GraphQL→ JSON + CLI→ OpenClaw→ Agentic Web

4. Agent as Developer

OpenClaw's approach: treat the agent like a senior developer who reads docs, authenticates, and calls APIs:

# OpenClaw: thinking like a developer
$ az group list --output json          # Azure REST via CLI
$ curl -H "Authorization: Bearer $TOK" \
    https://graph.microsoft.com/v1.0/me # Direct Graph API call
$ gh pr list --json number,title        # GitHub CLI
$ kubectl get pods -o json              # K8s API via CLI
$ fabric pipeline run --workspace prod  # Fabric CLI

# No protocol adapters. No middleware.
# Just APIs and CLIs — the same tools you'd use.

5. Peter Steinberger & OpenClaw

Steinberger created OpenClaw as an open-source agentic AI agent. In early 2026 he joined OpenAI while insisting OpenClaw remain open-source under a non-profit foundation. Key design decisions:

  • Runs locally — your data stays on your machine
  • Works with any LLM — Claude, GPT, Gemini, local models
  • Calls APIs directly — REST, GraphQL, gRPC — no translation layer
  • Shells out to CLIs — az, gh, kubectl, curl — the Unix way
  • Skills via markdown — extensible, human-readable, version-controllable
  • Fully open-source — no vendor lock-in, community governed

6. Why This Matters for Platform Engineers

As a platform engineer working with Azure, IaC, and Fabric, the API-first approach is immediately practical:

  • Azure CLI + agents — az already exposes every Azure operation. Agents call it directly.
  • Fabric REST APIs — pipeline triggers, KQL queries, data quality — all callable via HTTPS
  • Infrastructure as Code — agents read Bicep, run az deployment what-if, and propose changes
  • Graph API for M365 — user management, Teams, SharePoint — standard OAuth + REST
  • Policy governance — az policy commands for compliance checks, remediation, exemptions

No new protocol servers to deploy. No middleware to maintain. The APIs are already there.

7. Open Standards Win

The internet wasn't built by one company. TCP/IP, HTTP, HTML, and CSS are all open standards. The result was the largest collaborative system in human history.

The agentic web follows the same pattern: REST APIs are universal, CLIs are composable, OAuth is standard, and OpenClaw is open-source. The best agent interface is the one that already works.
