Architecture Studio — C4 Models & ADRs

Summary: An interactive architecture communication tool inspired by Martin Fowler's emphasis on evolutionary architecture and Simon Brown's C4 model. Drill into system context, containers, and browse Architecture Decision Records (ADRs) that document the "why" behind platform decisions.

Why Architecture Communication Matters

"The fundamental job of architecture is to manage complexity, and the best tool we have for managing complexity is communication." — Martin Fowler

Most architecture diagrams rot. They're created in Visio or Lucidchart, exported as PNGs, and forgotten. The Architecture Studio takes a different approach: the diagrams are code. They live in ClojureScript data structures, render interactively, and evolve with the platform.

C4 Model — Zoom Levels

The C4 model (Context, Containers, Components, Code) by Simon Brown gives us a shared vocabulary for architecture diagrams. This demo implements two levels:

LevelShowsAudience
System ContextSystems and their relationships to actorsEveryone
ContainerDeployable units within a systemDevelopers & architects

Click any system in the context view to drill down into its containers. Each container shows its technology and responsibility.

Architecture Decision Records

"Architecture is about the important stuff — whatever that is." — Ralph Johnson (via Martin Fowler)

ADRs capture the decisions that shape a system over time. They follow Michael Nygard's lightweight format: Context → Decision → Consequences. The ADR browser in this demo showcases 7 real platform decisions:

ADRDecisionStatus

Implementation

The Studio is built in pure ClojureScript with no framework. State management uses simple atoms — Rich Hickey's "just use data" philosophy. The C4 data is a vector of maps; drilling down filters by :system-id. ADR content is inline data that could be loaded from markdown files via a build step.

CSS uses the enterprise theme with custom properties for color, spacing, and motion tokens — following Kevin Powell's modern CSS approach. Every interactive element has prefers-reduced-motion support.
