1. Joiner-Mover-Leaver Lifecycle

Every workforce event — onboarding, transfer, or offboarding — triggers a cascade of identity and access changes. The JML system is built as an event-driven pipeline using Microsoft-native services: Entra ID Governance (Lifecycle Workflows) → Azure Functions → Service Bus → Entra Provisioning. Three Azure Functions handle event classification, workflow orchestration, and account provisioning respectively.

HRIS Event→ Lifecycle Workflows→ Event Listener (Function)→ Service Bus Topic→ Entra Provisioning
Why Service Bus over Event Hub? Service Bus is the right choice for JML: it's a message broker with topics/subscriptions, exactly-once delivery, dead-letter queues, and per-message acknowledgment. Event Hub is designed for high-throughput streaming (millions of events/sec) — overkill for JML where each hire/termination event must be individually processed and guaranteed.

Event Classification

The Event Listener (Function App #1) receives HRIS events, classifies them, normalizes worker details, and publishes to the Service Bus topic. Downstream consumers filter by classification.

Onboard (Joiner)

HRIS Event→ Lifecycle Workflow→ Service Bus→ Entra Provisioning→ AD Sync→ Groups + License

When HR submits a new hire, the Event Listener normalizes 15+ fields (name, department, manager, employee type, work location, exempt status) and publishes to the Service Bus topic. The Entra Provisioning function routes to the appropriate method — most commonly, SCIM 2.0 bulk upload for Hybrid AD provisioning via an on-premises provisioning agent. Post-provisioning steps include security group assignment, M365 license allocation, and Intune device enrollment.

Transfer (Mover)

HRIS Change Event→ Classify CHANGE→ Access Review→ Update Groups→ Update Packages

Change events for worker type, job title, manager, department, and position are classified and trigger transfer workflows. The risk is access accumulation: over time, transferred employees retain permissions from previous roles. Entra ID access reviews and time-bound access packages mitigate this.

Offboard (Leaver)

HRIS Termination→ Lifecycle Workflow→ Disable Sign-in→ Revoke Access→ Reclaim Licenses→ Archive

Entra ID Lifecycle Workflows can automate offboarding: disable sign-in, revoke active sessions, remove group memberships, and trigger a Logic App for mailbox conversion and OneDrive archival. License reclamation is the most common source of cost waste — a Copilot add-on at $30/user/month left on a disabled account for six months costs $180 for nothing.

3. Entra ID Security Groups

Access is always granted via team membership — never individual grants. Entra security groups map to role boundaries: an engineer's group membership determines their Fabric workspace access, GitHub team assignment, and Azure subscription visibility. B2B guest users are provisioned in Entra ID and added to the GitHub organization through team membership.

Confluence permissions are mapped to Entra security groups. The Copilot architecture uses Entra ID delegated permissions and OBO (On-Behalf-Of) flow, ensuring that agents only return data the user has access to — security-trimmed search results, not unfiltered indexes.

4. Intune Compliance

Device compliance policies enforce security baselines at the endpoint level. Intune evaluates devices against policies for encryption, OS version, antivirus status, and screen lock. Non-compliant devices are blocked from accessing M365 resources via Conditional Access policies — the enforcement boundary is Entra ID, not the device itself.

The SPA models compliance policies with percentage bars showing fleet-wide compliance. In production, the key insight is that compliance is not binary: a device can be "compliant" for encryption but "non-compliant" for OS version. Conditional Access policies can be configured to require specific compliance claims, not just overall compliance status.

5. Microsoft 365 Copilot Agents

Custom agents extend Microsoft 365 Copilot with domain-specific capabilities. Each agent is accessible directly from the M365 Copilot chat interface at m365.cloud.microsoft/chat.

AgentCapabilityAccess
Confluence Cloud Searches and summarizes Confluence Cloud pages inside M365 Copilot. Find wiki articles, pull up runbook steps, or get page summaries without leaving the chat window. Basic Copilot Chat — no paid add-on required
Project Planner Interactive project scoping agent. Prompt: "Interview me until you have 95% about what I actually want, not what I think I should want." Basic Copilot Chat
Teams Meetings Recaps Teams meeting transcripts, extracts action items and decisions. Search across past meeting notes for specific topics. Basic Copilot Chat
Works with basic Copilot Chat — no paid Microsoft 365 Copilot add-on license required. These agents are accessible to anyone with a standard M365 subscription that includes Copilot Chat. The paid Copilot add-on ($30/user/month) is required for the full work-grounded experience across Teams and Microsoft 365 apps, with eligible base licenses including M365 E3/E5 or O365 E3/E5.

How to Add Agents

  1. Direct link (quickest): Click any agent link above — it opens directly in the Copilot chat interface. No installation needed.
  2. Agent store: Open M365 Copilot → click Get agents → search by name → click the agent tile.
  3. Microsoft Teams: Open the Copilot app in Teams sidebar → Get agents → browse or search.

Agents remember context within a conversation. Tag an agent with @AgentName to direct a question to a specific agent mid-conversation. Pin frequently used agents for quick access.

7. Architecture Integration

The M365 Copilot agents connect to a broader platform architecture. The Foundry agent uses Entra ID token authentication, Microsoft Graph for user context, Jira for work item queries, and Fabric SQL for business data. Data sources span SharePoint, JIRA, Teams calls, Azure portal, Grafana, and Power BI.

The Confluence Cloud connector for Microsoft 365 Copilot enables the Confluence agent to index and search Confluence pages. The Graph connector powers natural-language queries over security alerts and operational data.

Entra ID (Auth)→ Microsoft Graph→ Foundry Agent→ Confluence / Jira / Fabric→ M365 Copilot Chat
